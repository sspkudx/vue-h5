# 技能使用规范

本文档描述了使用 vue-h5 项目中 AI 智能开发技能的最佳实践和规范。

## 技能调用约定

### 1. 明确的应用/包名称
- 名称应符合 npm 包名规范（小写字母、数字、连字符）
- 使用有意义的名称，避免使用通用词汇
- 示例：`my-app`、`user-portal`、`admin-panel`

### 2. 可选参数指定
- 端口号：`端口号设为 8080`
- 包类型：`类型为组件库`
- 包描述：`描述为 '用户认证相关工具函数'`
- 示例：`"创建名为 admin-panel 的应用，端口号设为 8080"`

### 3. 语义化描述
- 尽可能提供清晰的意图描述
- 说明应用的用途或包的功能
- 示例：`"创建名为 auth-helpers 的工具函数集，用于用户认证相关的工具函数"`

## 交互式创建流程

当使用技能时，系统会按以下流程进行交互：

### 1. 确认意图
- 确认用户要创建应用还是包
- 确认应用/包的名称和类型

### 2. 收集信息
- 询问必要的参数：
  - 应用/包名称
  - 端口号（对于应用）
  - 包类型（对于包）
  - 包描述（可选）
- 验证输入参数格式

### 3. 验证输入
- 检查名称是否符合规范
- 检查端口是否可用（对于应用）
- 检查目标目录是否已存在
- 验证依赖包的兼容性

### 4. 创建文件
- 生成完整的目录结构
- 创建配置文件（package.json, tsconfig.json, vue.config.js 等）
- 生成示例代码和组件
- 配置构建工具和开发工具

### 5. 验证结果
- 检查生成的文件是否正确
- 验证配置文件的语法
- 确保依赖关系正确配置
- 测试基本功能是否可用

### 6. 后续建议
- 提供安装依赖的指令
- 提供启动开发服务器的指令
- 提供构建和测试的指令
- 建议后续开发步骤

## 技能使用示例

### 创建 Vue 应用
```bash
# 基础用法
"创建名为 my-app 的 Vue 应用"

# 指定端口号
"创建名为 admin-panel 的应用，端口号设为 8080"

# 完整描述
"在 apps 目录下添加新的应用 user-portal，端口使用 3000"
```

### 创建依赖包
```bash
# 创建工具库
"创建名为 utils 的工具库包"

# 创建组件库
"创建名为 ui-components 的组件库，类型为组件库"

# 创建工具函数集（带描述）
"创建名为 auth-helpers 的工具函数集，描述为 '用户认证相关工具函数'"

# 创建插件库
"创建名为 vue-plugins 的插件库，用于 Vue 插件开发"
```

## 参数说明

### 应用创建参数
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 名称 | string | 必填 | 应用名称，符合 npm 包名规范 |
| 端口号 | number | 自动分配 | 开发服务器端口，可选参数 |

### 包创建参数
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 名称 | string | 必填 | 包名称，符合 npm 包名规范 |
| 类型 | enum | 工具库 | 包类型：工具库、组件库、工具函数集、插件库 |
| 描述 | string | 空 | 包的简要描述，可选参数 |

## 技能配置

### 技能文件位置
技能文件位于以下目录：
- 项目技能目录：`/skills/`
- AI 编辑器技能目录：
  - CatPaw: `.catpaw/skills/`
  - Cursor: `.cursor/skills/`
  - Windsurf: `.windsurf/skills/`
  - Trae: `.trae/skills/`

### 技能文件格式
每个技能包含一个 `SKILL.md` 文件，包含：
- 元数据头（name, description）
- 技能概述和使用场景
- 详细使用说明和示例
- 配置参数说明
- 错误处理和故障排除

## 最佳实践

### 1. 命名规范
- 使用小写字母、数字和连字符
- 避免使用特殊字符和下划线
- 保持名称简洁且有描述性

### 2. 端口选择
- 避免使用常用端口（如 80, 443, 3000, 8080）
- 检查端口是否被占用
- 建议使用 3000 以上的端口

### 3. 包类型选择
- **工具库**：纯工具函数，无框架依赖
- **组件库**：Vue 组件，需要 Vue 3.x 依赖
- **工具函数集**：业务相关的工具函数
- **插件库**：Vue 插件或通用插件

### 4. 版本管理
- 新创建的应用/包版本默认为 `1.0.0`
- 遵循语义化版本规范
- 及时更新依赖包版本

## 故障排除

### 常见问题

#### 1. 技能未识别
- 确保技能文件放在正确的目录
- 检查技能文件的格式和内容
- 重启 AI 编辑器

#### 2. 创建失败
- 检查名称是否符合规范
- 确保目标目录不存在
- 检查是否有足够的权限

#### 3. 依赖安装失败
- 检查网络连接
- 清理并重新安装依赖
- 更新 npm/yarn/pnpm 版本

#### 4. 启动失败
- 检查端口是否被占用
- 验证配置文件语法
- 查看错误日志
